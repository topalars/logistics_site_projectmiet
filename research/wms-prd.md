# Product Requirements Document (PRD)
## Система управления складом (WMS) - "LogiCore WMS"

**Версия:** 1.0  
**Дата:** 2026-01-01  
**Статус:** Draft  

---

## 1. Введение

### 1.1 Цель продукта
Создание современной веб-системы управления складом (WMS), которая оптимизирует логистические процессы, повышает точность учета товаров и ускоряет обработку заказов по стандартам лидеров рынка (Ozon, Wildberries, Amazon).

### 1.2 Целевая аудитория
- **Складской персонал:** кладовщики, комплектовщики, приемщики
- **Менеджеры склада:** начальники смен, супервайзеры
- **Администраторы:** управление пользователями, настройка системы
- **Аналитики:** мониторинг KPI, отчетность
- **Интеграторы:** подключение внешнего оборудования и систем

### 1.3 Основные проблемы, которые решает система
- Низкая скорость обработки заказов
- Ошибки при комплектации и отгрузке
- Отсутствие реальной видимости остатков
- Неэффективное использование складского пространства
- Сложности интеграции с маркетплейсами и ERP

## 2. Функциональные требования

### 2.1 Модуль "Приемка товаров" (Inbound)

#### 2.1.1 Создание заданий на приемку
- Импорт ожидаемых поставок из внешних систем (ERP, маркетплейсы)
- Ручное создание заданий на приемку
- Привязка к заказу поставщика / накладной
- Планирование времени прибытия транспорта

#### 2.1.2 Процесс приемки
- Сканирование штрихкодов товаров (поддержка 1D/2D кодов)
- Сверка фактического количества с плановым
- Регистрация расхождений (недостача, излишки, брак)
- Фотофиксация повреждений (опционально)
- Печать этикеток для маркировки товара
- Поддержка серийных номеров и сроков годности

#### 2.1.3 Контроль качества
- Выборочная или полная проверка качества
- Статусы товара: "Принят", "На проверке", "Брак", "Карантин"
- Регистрация причин брака

### 2.2 Модуль "Размещение товаров" (Put-away)

#### 2.2.1 Алгоритмы размещения
- Автоматическое определение оптимальной ячейки на основе:
  - Габаритов товара
  - Оборотности (ABC-анализ)
  - Совместимости товаров (температурный режим, опасность)
  - Близости к зонам комплектации
- Поддержка многоуровневого хранения
- Динамическое размещение (без фиксированных мест)

#### 2.2.2 Задания на размещение
- Формирование маршрутных листов для кладовщиков
- Приоритизация задач по срочности
- Отслеживание прогресса в реальном времени
- Подтверждение размещения через сканирование

#### 2.2.3 Управление ячейками
- Статусы ячеек: "Свободна", "Занята", "Зарезервирована", "Неисправна"
- Визуализация карты склада
- История перемещений по ячейкам

### 2.3 Модуль "Управление запасами" (Inventory)

#### 2.3.1 Учет товаров
- Хранение информации о товаре: артикул, название, категория, габариты, вес
- Поддержка единиц измерения (шт, кг, м, упак)
- Учет по партиям (FIFO, FEFO, LIFO)
- Отслеживание сроков годности
- Серийные номера и уникальные идентификаторы

#### 2.3.2 Инвентаризация
- Плановая и внеплановая инвентаризация
- Частичная инвентаризация по зонам/категориям
- Циклический подсчет (cycle counting)
- Мобильный интерфейс для счетчиков
- Автоматическая сверка и корректировка остатков
- Отчеты по расхождениям

#### 2.3.3 Перемещения внутри склада
- Перемещение между ячейками
- Перемещение между зонами
- Консолидация остатков
- Задания на перемещение с маршрутизацией

#### 2.3.4 Резервирование
- Автоматическое резервирование под заказы
- Приоритеты резервирования
- Срок действия резерва
- Освобождение резерва

### 2.4 Модуль "Комплектация заказов" (Order Picking)

#### 2.4.1 Стратегии комплектации
- **Дискретная:** один заказ - один комплектовщик
- **Волновая:** группировка заказов по времени/направлению
- **Зонная:** разные зоны склада - разные исполнители
- **Пакетная:** несколько заказов одновременно (batch picking)
- **Волны + Зоны:** комбинированный подход

#### 2.4.2 Оптимизация маршрутов
- Построение оптимального пути по складу
- Избегание пересечений маршрутов
- Учет габаритов и веса при формировании маршрута

#### 2.4.3 Процесс комплектации
- Маршрутные листы с пошаговой навигацией
- Сканирование для подтверждения взятия товара
- Контроль количества (предотвращение ошибок)
- Подсветка ячеек (интеграция с Pick-to-Light)
- Печать этикеток для коробок
- Взвешивание и контроль веса

#### 2.4.4 Упаковка
- Рекомендации по упаковке на основе габаритов
- Выбор типа тары (коробка, пакет, паллет)
- Печать транспортных накладных
- Фотофиксация упаковки (опционально)

### 2.5 Модуль "Отгрузка" (Outbound)

#### 2.5.1 Подготовка к отгрузке
- Формирование отгрузочных партий
- Группировка по перевозчикам/направлениям
- Проверка комплектности перед отгрузкой

#### 2.5.2 Погрузка
- Задания на погрузку с указанием места в транспорте
- Сканирование при погрузке
- Контроль последовательности загрузки

#### 2.5.3 Документооборот
- Генерация транспортных накладных (CMR, ТТН)
- Электронные подписи
- Интеграция с системами перевозчиков

### 2.6 Модуль "Управление заказами" (Order Management)

#### 2.6.1 Импорт заказов
- Интеграция с маркетплейсами (Ozon, WB, Яндекс.Маркет)
- Интеграция с интернет-магазинами (API, CSV, XML)
- Интеграция с ERP-системами (1С, SAP)
- Ручное создание заказов

#### 2.6.2 Обработка заказов
- Валидация заказов (проверка наличия, адресов)
- Приоритизация (VIP, срочные, обычные)
- Консолидация заказов одного клиента
- Разделение заказов при частичной доступности

#### 2.6.3 Статусы заказов
- "Новый", "В обработке", "Комплектуется", "Упакован", "Отгружен", "Доставлен", "Отменен", "Возврат"

### 2.7 Модуль "Аналитика и отчетность"

#### 2.7.1 Дашборды
- Оперативная сводка по складу
- KPI в реальном времени
- Графики и диаграммы
- Фильтрация по периодам, зонам, сотрудникам

#### 2.7.2 Ключевые метрики (KPI)
- **Точность инвентаризации:** % расхождений
- **Скорость приемки:** товаров/час
- **Скорость комплектации:** заказов/час, строк/час
- **Время цикла заказа:** от получения до отгрузки
- **Использование пространства:** % заполненности ячеек
- **Производительность сотрудников:** заданий/смену
- **Процент брака:** % от общего объема
- **On-time shipment:** % отгруженных вовремя заказов

#### 2.7.3 Стандартные отчеты
- Отчет по остаткам (на дату, движение)
- Отчет по оборачиваемости товаров
- Отчет по выполнению заказов
- Отчет по производительности сотрудников
- Отчет по расхождениям при инвентаризации
- Отчет по браку и возвратам
- ABC/XYZ анализ товаров

#### 2.7.4 Экспорт данных
- Форматы: Excel, CSV, PDF
- Планирование автоматической рассылки отчетов
- API для выгрузки данных

### 2.8 Модуль "Администрирование"

#### 2.8.1 Управление пользователями
- Ролевая модель (RBAC)
- Предопределенные роли: Администратор, Менеджер, Кладовщик, Комплектовщик, Аналитик
- Настройка прав доступа к функциям и данным
- Аудит действий пользователей

#### 2.8.2 Настройка склада
- Конфигурация зон склада (приемка, хранение, комплектация, отгрузка)
- Настройка ячеек и стеллажей
- Определение типов хранения (паллетное, коробочное, мелкоячеистое)
- Настройка правил размещения и комплектации

#### 2.8.3 Справочники
- Товары и категории
- Контрагенты (поставщики, клиенты, перевозчики)
- Единицы измерения
- Типы упаковок
- Причины брака и расхождений

#### 2.8.4 Интеграции
- Настройка подключений к внешним системам
- Маппинг данных
- Мониторинг статуса интеграций
- Логирование ошибок

## 3. Нефункциональные требования

### 3.1 Производительность
- Время отклика интерфейса: < 200 мс для стандартных операций
- Поддержка одновременной работы: 100+ пользователей
- Обработка заказов: 1000+ заказов в час
- Время загрузки дашборда: < 2 секунд

### 3.2 Масштабируемость
- Горизонтальное масштабирование серверной части
- Поддержка нескольких складов в одной системе
- Возможность добавления новых модулей без остановки системы

### 3.3 Надежность
- Доступность системы: 99.5% (SLA)
- Автоматическое резервное копирование данных (ежедневно)
- Восстановление после сбоев: < 1 часа
- Транзакционная целостность операций

### 3.4 Безопасность
- HTTPS для всех соединений
- Аутентификация через OAuth 2.0 / JWT
- Двухфакторная аутентификация (опционально)
- Шифрование чувствительных данных
- Соответствие 152-ФЗ (для РФ)
- Регулярные аудиты безопасности

### 3.5 Совместимость
- **Браузеры:** Chrome, Firefox, Safari, Edge (последние 2 версии)
- **Мобильные устройства:** iOS 14+, Android 10+
- **Оборудование:** сканеры штрихкодов (Zebra, Honeywell), терминалы сбора данных, принтеры этикеток

### 3.6 Локализация
- Русский язык (основной)
- Английский язык (опционально)
- Поддержка форматов даты/времени РФ
- Валюта: RUB (основная), USD, EUR

## 4. Технические требования

### 4.1 Архитектура
- **Фронтенд:** React 18+ с TypeScript
- **Бэкенд:** Node.js (NestJS) или Python (FastAPI)
- **База данных:** PostgreSQL (основная), Redis (кэш)
- **Очереди сообщений:** RabbitMQ или Apache Kafka
- **Контейнеризация:** Docker, Kubernetes
- **CI/CD:** GitLab CI или GitHub Actions

### 4.2 API
- RESTful API для основных операций
- GraphQL для сложных запросов с агрегацией
- WebSocket для real-time обновлений
- OpenAPI/Swagger документация

### 4.3 Интеграции
- **Маркетплейсы:** Ozon API, Wildberries API, Яндекс.Маркет API
- **ERP:** 1С Предприятие (COM/API), SAP (RFC/IDoc)
- **Перевозчики:** СДЭК, Boxberry, Почта России API
- **Оборудование:** Zebra SDK, Honeywell SDK
- **Стандарты:** EDI (EDIFACT, X12), XML, JSON

### 4.4 Мониторинг и логирование
- **Мониторинг:** Prometheus + Grafana
- **Логирование:** ELK Stack (Elasticsearch, Logstash, Kibana)
- **Трейсинг:** Jaeger или Zipkin
- **Алертинг:** Telegram, Email, PagerDuty

### 7.1 Функциональные критерии
- [ ] Все модули работают согласно спецификации

- [ ] Количество ошибок при комплектации снижено на 80%

## 8. Риски и зависимости

### 8.1 Технические риски
- **Интеграции с внешними API:** нестабильность сторонних сервисов
  - *Митигация:* Кэширование, повторные попытки, очередь сообщений
- **Производительность при росте данных:** деградация скорости
  - *Митигация:* Партиционирование БД, индексы, кэширование
- **Оборудование:** несовместимость сканеров/принтеров
  - *Митигация:* Тестирование на популярных моделях, абстракция устройств

### 8.2 Бизнес-риски
- **Сопротивление персонала:** нежелание менять процессы
  - *Митигация:* Обучение, постепенное внедрение, обратная связь
- **Изменение требований:** новые функции в процессе разработки
  - *Митигация:* Гибкая методология, приоритизация бэклога
- **Зависимость от ключевых разработчиков:** уход специалистов
  - *Митигация:* Документация, код-ревью, кросс-обучение

### 8.3 Зависимости
- Доступ к API маркетплейсов (требуется регистрация и одобрение)
